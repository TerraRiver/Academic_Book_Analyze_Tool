
### **项目名称：TR_book_read_tool (国际关系研究阅读助手)**

### 1. 项目目标

开发一款桌面应用程序，旨在帮助国际关系研究人员快速阅读和分析书籍类PDF文献。程序将自动化处理从PDF章节化、文本解析、LLM总结到生成结构化Word报告的全过程，大幅提升研究效率。

### 2. 核心设计原则

* **模块化 (Modularity)** ：各功能模块（UI、文件处理、API调用）高度解耦，便于独立开发、测试和维护。
* **用户体验 (User Experience)** ：界面直观，提供清晰的状态反馈。通过异步处理避免UI卡顿，确保操作流畅。
* **健壮性 (Robustness)** ：具备完善的错误处理和日志记录机制，能优雅地处理文件错误、网络异常和API调用失败。
* **可配置性 (Configurability)** ：核心参数（如API密钥、LLM Prompt）应易于用户在UI中修改，以适应不同需求。

### 3. 技术选型 (Python)

* **GUI框架** :
* **推荐** : `PySide6` (或 `PyQt6`) - 功能强大，支持丰富的UI组件和信号/槽机制，非常适合构建复杂的桌面应用。
* **备选** : `CustomTkinter` - 如果追求更现代、简洁的UI风格且功能需求相对简单，这是一个不错的选择。
* **PDF处理** : `pypdf` - 用于读取PDF书签、页数信息以及按页码范围分割PDF。
* **HTTP请求 (API调用)** : `httpx` - 同时支持同步和异步请求，是实现并发API调用的理想选择。
* **Word文档生成** : `python-docx` - 用于创建和操作 `.docx`文件，支持样式、表格和图片。
* **并发处理** : `asyncio` - Python内置的异步I/O框架，用于管理并发的API请求。
* **依赖管理** : `pip` 配合 `requirements.txt` 或使用 `Poetry`。

### 5. 开发模块分解 (Step-by-Step)

#### 模块一：项目初始化与UI壳体

1. **目标** : 搭建项目骨架，创建可运行的主窗口。
2. **实现** :

* 设置Python虚拟环境，安装 `PySide6`。
* 创建 `main.py`作为程序入口。
* 在 `ui/main_window.py`中，设计主窗口布局：
  * 左侧：一个 `QListWidget`用于显示书籍列表。
  * 右侧：一个占位的 `QWidget`，用于后续显示书籍详情。
* 实现 `core/book_manager.py`的初步功能：能够扫描 `data`目录，并将找到的书籍（通过 `metadata.json`）加载到UI左侧的列表中。

1. **产出** : 一个可以启动的空窗口，能够显示 `data`目录下的书籍。

#### 模块二：章节管理功能

1. **目标** : 实现PDF书签读取和手动输入章节信息的功能。
2. **实现** :

* 在 `core/pdf_processor.py`中，创建 `get_bookmarks(pdf_path)`函数，使用 `pypdf`读取书签，返回章节名和页码。
* 在UI中，为右侧区域设计一个“章节管理”界面。当用户选择一本书后：
  * 自动调用 `get_bookmarks`并填充一个 `QTableWidget`。
  * 如果无书签，则表格为空，用户可以通过“添加章节”按钮手动输入章节名、起止页码。
  * 提供删除按钮，允许用户删除不需要分析的章节。
* 用户点击“保存章节”后，将章节信息写入对应书籍的 `metadata.json`文件，并更新状态为 `chaptering`。

1. **产出** : 用户可以为PDF定义需要处理的章节，并将这些信息持久化。

#### 模块三：异步API处理器与文件处理

1. **目标** : 构建核心的后台处理引擎，并发执行所有耗时任务。
2. **实现** :

* **核心逻辑在 `core/api_handler.py`** 。
* **主控制函数** : 创建一个 `process_book(book_metadata)`异步函数。
* **并发任务** : 在 `process_book`中，使用 `asyncio.gather`为 `metadata.json`中每个状态为 `pending`的章节创建一个处理任务。
* **单个章节处理任务 `process_chapter(chapter_info)`** :
  1.  **（可选）切分PDF** : 如果需要，调用 `pdf_processor.py`中的函数切分出小PDF。
  1.  **调用MinerU** : 发起异步 `httpx`请求到MinerU API。处理成功后，保存JSON，更新 `metadata.json`中该章节的状态和路径。
  1.  **调用DeepSeek** : 读取上一步的解析结果，格式化为文本，结合 `config.ini`中的Prompt，发起异步 `httpx`请求到DeepSeek API（ **强烈推荐使用JSON模式** ）。保存分析结果，再次更新 `metadata.json`。
* **UI反馈** : 使用PySide的信号(signal)机制。后台任务在状态更新时（如“开始解析第2章”、“第2章解析完成”），发射一个信号，主UI线程中的槽函数(slot)接收到信号后，刷新书籍列表的图标和进度条。

1. **产出** : 一个非阻塞的后台处理系统，能够高效地完成从PDF解析到LLM分析的全过程。

#### 模块四：Word报告生成

1. **目标** : 将所有章节的分析结果整合成一份精美的Word报告。
2. **实现** :

* 在 `core/report_generator.py`中，创建 `create_report(book_metadata)`函数。
* 使用 `python-docx`加载 `assets/report_template.docx`模板。
* 遍历 `book_metadata`中的所有 `complete`状态的章节。
* 对于每个章节，读取其 `chapters_analysis`中的分析结果（JSON文件）。
* 将JSON中的内容（核心论点、论据、概念等）按预设格式写入Word文档，使用模板中的样式（如 `Heading 2`用于章节标题，`Quote`用于原文引用）。
* 添加一个封面页，包含书名和作者。
* 保存最终的Word文档到书籍目录下。
* UI上提供一个“生成报告”的按钮，当书籍状态为 `complete`时激活。

1. **产出** : 一键生成结构清晰、格式专业的 `.docx`研究报告。

#### 模块五：设置页面

1. **目标** : 提供一个界面，让用户可以配置API密钥和自定义Prompt。
2. **实现** :

* 在 `ui/settings_dialog.py`中创建一个 `QDialog`。
* 界面包含输入框用于填写 `MinerU_API_KEY`和 `DEEPSEEK_API_KEY`。
* 包含一个大的文本编辑框，用于编辑LLM分析时使用的Prompt。
* 程序启动时读取 `config.ini`文件加载配置。用户点击“保存”时，将UI上的内容写回 `config.ini`。

1. **产出** : 用户可以不修改代码，方便地配置个人凭证和优化分析指令。

### 6. 开发路线图 (Phased Roadmap)

* **第一阶段：本地核心功能 (MVP - Minimum Viable Product)**
  1. 完成**模块一**和 **模块二** ：实现UI基础和章节管理。
  2. 创建 **模块四** （报告生成）的“桩”版本：用**虚拟的/手写的**分析JSON文件来生成Word报告。
  3. **目标** : 验证本地文件操作和报告生成流程的通畅。
* **第二阶段：集成API与异步处理**
  1. 完成 **模块三** ：实现完整的异步API调用逻辑。
  2. 在UI中集成后台任务的启动和状态反馈。
  3. **目标** : 打通端到端的自动化处理流程，这是项目的核心价值。
* **第三阶段：完善与优化**
  1. 完成 **模块五** ：实现设置页面。
  2. 精细化错误处理：为不同类型的错误（网络、API限流、文件找不到）提供明确的用户提示。
  3. UI美化：优化布局，添加图标，增加进度条等视觉元素。
  4. 打包发布：使用 `PyInstaller`或 `cx_Freeze`将程序打包成可执行文件，方便分发。
